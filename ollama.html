Recommended Solutions

Solution 1: Enable CORS on Your Ollama Server

This is the most reliable solution. Add CORS headers to your Ollama server configuration:

# On your server, set these environment variables export OLLAMA_ORIGINS="*" export OLLAMA_HOST="0.0.0.0:11434" # Then restart Ollama sudo systemctl restart ollama

Solution 2: Use a Reverse Proxy

Set up a reverse proxy (like Nginx) to handle CORS for you:

# Sample Nginx configuration server { listen 80; server_name your-domain.com; location /ollama/ { proxy_pass http://139.99.69.205:11434/; # Add CORS headers add_header 'Access-Control-Allow-Origin' '*' always; add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS' always; add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range' always; add_header 'Access-Control-Expose-Headers' 'Content-Length,Content-Range' always; if ($request_method = 'OPTIONS') { add_header 'Access-Control-Allow-Origin' '*'; add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS'; add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range'; add_header 'Access-Control-Max-Age' 1728000; add_header 'Content-Type' 'text/plain; charset=utf-8'; add_header 'Content-Length' 0; return 204; } } }

Solution 3: Browser Extension (Temporary Fix)

Install a CORS-unblocking browser extension for development:

  • CORS Unblock for Chrome
  • CORS Everywhere for Firefox

Note: This is only for development, not production use.

Advanced Configuration

Custom Proxy Solution

If you can't modify the server, you can set up a custom CORS proxy:

// Simple Node.js CORS proxy const express = require('express'); const cors = require('cors'); const fetch = require('node-fetch'); const app = express(); const PORT = 3001; app.use(cors()); app.use(express.json()); app.post('/proxy/ollama', async (req, res) => { try { const response = await fetch('http://139.99.69.205:11434/api/chat', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(req.body) }); const data = await response.json(); res.json(data); } catch (error) { res.status(500).json({ error: error.message }); } }); app.listen(PORT, () => { console.log(`CORS proxy running on port ${PORT}`); });

Install required packages: npm install express cors node-fetch

Security Considerations

If you're dealing with a 403 error, your server might have IP restrictions:

  1. Check your server's firewall settings
  2. Verify that your IP isn't blocked
  3. Check if your Ollama server requires authentication
  4. Ensure your server isn't restricting certain user agents

Need more help? Check the Ollama documentation for specific guidance.
